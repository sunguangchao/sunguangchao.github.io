如何利用Hexo和Github建立属于自己的博客

今天终于把这个Blog给搞好了，前面一直纠结于Jelly+Github的模式，配置环境太过于麻烦而放下了一段时间，最近有空就看看，就给弄好了。这里推荐的Hexo+Github的模式就好很多。

##名词解释

###What is Hexo?
官方是这样解释的：Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.
附上官方网址：Hexo.

###What is Github?
不知道的自行google（自备梯子-Lantern）

##环境配置

###git
这里默认已经安装好git并且已有Github的账号，SSH已经配置好（挖个坑，后面再填）。

###Node.js
官网地址：Node.js
Node.js简介：Node.js is a JavaScript runtime built on Chrome’s V8 JavaScript engine. Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient. Node.js’ package ecosystem, npm, is the largest ecosystem of open source libraries in the world.
下载好Node.js一路默认安装就好。

##安装Hexo
上面的步骤全部完成之后，下一步就是安装Hexo：

$ npm install -g hexo-cli

安装完成后，进行初始化。建立一个文件夹命名为”Hexo”。
在那个文件夹中打开Git Bash，输入以下命令：

$ hexo init 
$ npm install

初始化完成之后，可以看到目录结构:

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

各个部分都有其作用（再挖个坑），比较重要的是_config.yml这个文件。

##配置更改
在_config.yml这个文件中，需要将deploy下面改成这个形式：

deploy:
  type: git
  repo: <repository url>
  branch: [branch]
  message: [message]

特别注意的是”：”之后必须有一个空格，不然肯定是错的。改好之后就为后面发布自己的Blog做好了准备。

##新建一篇文章

$ hexo new [layout] <title>

使用这个命令可以新建一个markdown文件，在Hexo\source’\’_posts根目录下
直接进行编辑，就是一片文章。

##本地测试
在Hexo3中server和主体结构动画是分开的，所以需要安装server。

$ npm install hexo-server --save

安装之后用以下命令可以运行server：

$ hexo server

一般来说在浏览器中输入http://localhost:4000就可以看到Blog的页面。
我这里的4000端口被占用了，可以用以下命令来改变端口：

$ hexo server -p 5000

##发布博客
先安装hexo-deployer-git：

$ npm install hexo-deployer-git --save

三步走：

$ hexo clean
$ hexo generate
$ hexo deploy
